# Workflow v0.1.2 Feature Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add practical usability improvements for `open`, `list`, `new`, and shell lint automation, then publish `v0.1.2`.

**Architecture:** Keep the toolkit simple and shell-friendly. Add a script-safe `--path` mode to `open`, improve human-readable default output, format `list` as a lightweight text table, support an explicit `--slug` option in `new`, and add a minimal GitHub Actions workflow for shellcheck.

**Tech Stack:** Bash, Markdown, GitHub Actions

---

## Scope

1. `open`: default summary mode + `--path`
2. `list`: table-like output
3. `new`: `--slug <value>` support
4. GitHub Actions shellcheck workflow
5. Update version, changelog, and release to `v0.1.2`
