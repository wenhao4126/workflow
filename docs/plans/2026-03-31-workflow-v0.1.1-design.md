# Workflow v0.1.1 Open Output Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Improve `workflow.sh open` output to be more human-friendly while keeping behavior simple and backward compatible.

**Architecture:** Keep `open-task.sh` file-system based. Instead of printing only the matched path, print a short summary containing the task directory, available task files, and status from `TASK.md` when available. Then bump version metadata and document the change in `CHANGELOG.md`.

**Tech Stack:** Bash, Markdown
