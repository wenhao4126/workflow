# Workflow v0.1.4 Feature Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Make `workflow.sh` the consistent public CLI entrypoint for newly added features, add `new --json`, extend doctor checks, add plain/pretty output modes, and ship a real PNG demo asset in `v0.1.4`.

**Architecture:** Keep the implementation shell-first and dependency-light. Reuse the existing task scripts, extend them with small output flags, and expose the features consistently through `workflow.sh`. Use a lightweight generated demo image file for documentation rather than introducing screen recording workflows.

**Tech Stack:** Bash, Markdown, PNG asset generation

---

## Scope

1. `workflow.sh` entrypoint support/documentation for new flags
2. `new --json`
3. `doctor` checks `.github/workflows/shellcheck.yml`
4. `open` / `list` support `--pretty` and `--plain`
5. Add real demo PNG and reference it in README
6. Release `v0.1.4`
