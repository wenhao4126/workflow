# Workflow v0.1.3 Feature Design

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add script-friendly JSON output, status control/filtering, a detailed task view, and lightweight README demo assets, then publish `v0.1.3`.

**Architecture:** Extend the existing Bash scripts conservatively. Add opt-in flags for JSON output and detailed views, keep the default human-readable output intact, and avoid introducing external JSON tooling. README demo assets should be static files under a docs/demo-style directory.

**Tech Stack:** Bash, Markdown, plain-text CLI output

---

## Scope

1. `open --json`
2. `new --status`
3. `list --status` (plus optional `--json` to support script integration cleanly)
4. `open --detail`
5. README static demo image placeholder/reference
6. Release `v0.1.3`
